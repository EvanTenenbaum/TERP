# MCP Server Configuration

This project uses MCP (Model Context Protocol) servers to enable AI agents to interact with external services.

## Setup Instructions

1. Copy `.mcp.json.example` to `.mcp.json`:

   ```bash
   cp .mcp.json.example .mcp.json
   ```

2. The example already includes the local `domscribe` MCP server with no secrets required.

3. If you want the DigitalOcean MCP server too, edit `.mcp.json` and replace `your_digitalocean_api_token_here` with your actual DigitalOcean API token.

4. The `.mcp.json` file is gitignored to prevent accidentally committing secrets.

## DigitalOcean MCP Server

The DigitalOcean MCP server enables AI agents to:

- View deployment status and logs
- Monitor applications on DigitalOcean
- Trigger deployments programmatically
- Check build status and errors
- Manage database connections
- View app health metrics

## Security

**IMPORTANT:** Never commit `.mcp.json` to version control. The Domscribe entry is safe, but DigitalOcean entries can contain API tokens.

## Domscribe MCP Server

The Domscribe MCP server lets Claude Code use the same local TERP UI-runtime inspection flow as Codex:

- query live rendered DOM/runtime state
- inspect wrapper components versus native elements
- use the browser overlay/annotation flow when the dev server is running
- verify a local UI change against the real browser output before broader QA

The actual configuration is stored in:

- Project-level: `.mcp.json` (gitignored)
- Claude Code global settings: `~/.claude/settings.json`
