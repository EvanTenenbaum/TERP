## Quick Wins

- INV-UX-001: Delete action remains clickable even when selected batches have on-hand inventory, causing reactive blocked errors after user intent is committed. -> Gate delete action pre-click using selection eligibility checks (onHandQty <= 0), disable destructive action when blocked rows exist, and expose blocked count with one-click 'Select eligible only'.
- INV-UX-002: Blocked delete emits both toast.error and persistent save-state error indicator, splitting attention and increasing cognitive load. -> Use contextual in-surface error banner for blocked delete; reserve toasts for success and undo confirmations only.
- INV-UX-008: The inventory work surface becomes visually busy during batch operations because filters, stats, selection actions, and error states compete in the same viewport at the same time. -> Introduce a focused selection mode: simplify visible chrome while rows are selected, keep all functions available via progressive disclosure, and prioritize the active batch task strip over secondary dashboard content.
- INV-UX-003: Blocked-delete message explains prerequisite actions but does not provide direct recovery controls in-context. -> Add actionable banner CTAs: 'Select eligible only', 'Adjust quantity', and 'Open Receiving'.
- INV-UX-005: Undo is toast-transient and can expire before keyboard and assistive-technology users can trigger it. -> Add persistent in-surface fallback control: 'Undo last delete' in status bar while undo window is active.
