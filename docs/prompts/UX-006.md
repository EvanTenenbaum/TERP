# UX-006: Add Error Recovery UI with Retry

## Implementation Guide

### Problem
Failed data loads show error messages but no retry mechanism. Users must manually refresh the page to try again.

### Affected Pages
- `Inventory.tsx` - No retry on failed load
- `ClientProfilePage.tsx` - No retry on transaction fetch failure
- Most pages with data fetching

### Solution
1. Create ErrorWithRetry component
2. Include refetch function in error display
3. Add exponential backoff for automatic retry

### Files to Create/Modify
- Create `client/src/components/common/ErrorWithRetry.tsx`
- Update affected pages

### Implementation
```typescript
// ErrorWithRetry.tsx
interface Props {
  error: Error;
  onRetry: () => void;
  isRetrying?: boolean;
}

export function ErrorWithRetry({ error, onRetry, isRetrying }: Props) {
  return (
    <div className="p-8 text-center">
      <AlertCircle className="mx-auto h-12 w-12 text-red-500" />
      <h3 className="mt-4 font-semibold">Failed to load data</h3>
      <p className="mt-2 text-muted-foreground">{error.message}</p>
      <Button onClick={onRetry} disabled={isRetrying} className="mt-4">
        {isRetrying ? (
          <>
            <Loader2 className="mr-2 h-4 w-4 animate-spin" />
            Retrying...
          </>
        ) : (
          <>
            <RefreshCw className="mr-2 h-4 w-4" />
            Try Again
          </>
        )}
      </Button>
    </div>
  );
}

// Usage
if (query.isError) {
  return <ErrorWithRetry error={query.error} onRetry={query.refetch} />;
}
```

### Acceptance Criteria
- Retry button on all error states
- Loading indicator during retry
- Clear error message displayed
