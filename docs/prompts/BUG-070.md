# BUG-070: Fix Silent Auth Failure in DashboardLayout

## Implementation Guide

### Problem
In `DashboardLayout.tsx` (lines 84-96), auth check failures silently set a default guest user without any notification. Users don't know if authentication actually failed.

### Current Code
```typescript
React.useEffect(() => {
  fetch("/api/auth/me")
    .then(res => res.json())
    .then(data => {
      if (data.user) {
        setUser(data.user);
      }
    })
    .catch(() => {
      // User not logged in - SILENT FAILURE
      setUser({ name: "Guest", email: "guest@example.com" });
    });
}, []);
```

### Solution
1. Show explicit error message when auth fails
2. Redirect to login on auth failure
3. Distinguish between network errors and unauthenticated state

### Files to Modify
- `client/src/components/DashboardLayout.tsx`

### Implementation
```typescript
.catch((error) => {
  console.error('Auth check failed:', error);
  toast.error('Session expired. Please log in again.');
  navigate('/login');
});
```

### Acceptance Criteria
- Users see clear message when auth fails
- Redirect to login on session issues
- Network errors distinguished from auth errors
