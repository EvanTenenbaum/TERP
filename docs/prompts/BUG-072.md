# BUG-072: Fix Silent localStorage Failure

## Implementation Guide

### Problem
In `Orders.tsx` (lines 65-70) and other pages, localStorage operations silently fail without user notification. If localStorage is full or disabled, users lose their filter state.

### Current Code
```typescript
try {
  localStorage.setItem(ORDERS_FILTER_KEY, JSON.stringify(filters));
} catch {
  // Silently fail
}
```

### Solution
1. Show toast warning when localStorage unavailable
2. Fall back to session storage or in-memory state
3. Log issue for debugging

### Files to Modify
- `client/src/pages/Orders.tsx`
- Other pages with similar patterns

### Implementation
```typescript
try {
  localStorage.setItem(ORDERS_FILTER_KEY, JSON.stringify(filters));
} catch (error) {
  console.warn('localStorage unavailable:', error);
  toast.warning('Filter preferences could not be saved', {
    description: 'Your filters will reset on page refresh'
  });
  // Fall back to sessionStorage
  try {
    sessionStorage.setItem(ORDERS_FILTER_KEY, JSON.stringify(filters));
  } catch {}
}
```

### Acceptance Criteria
- Users notified when persistence fails
- Graceful fallback to session storage
- No complete loss of functionality
