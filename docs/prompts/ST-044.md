# ST-044: Mega QA - Runbook (“Run the Mega QA” From Anywhere)

## Context

We need a canonical, self-contained workflow so any AI engineer can run Mega QA and interpret/replay results without tribal knowledge.

## Implementation Guide

- Create `docs/testing/MEGA_QA_RUNBOOK.md` that documents:
  - `pnpm mega:qa` (default)
  - `pnpm mega:qa:local` and `pnpm mega:qa:ci`
  - key flags (`--scenario`, `--journeys`, `--seed`, `--out-dir`, `--mode`)
  - where outputs are written and which file is the “entrypoint” (manifest)
- Document replay flow:
  - how to replay a single failure using bundle metadata
  - how to locate traces/screenshots/network evidence
- Document bug creation:
  - how to convert Mega QA output into roadmap bugs (via ST-043 integration)
- Add troubleshooting section for:
  - DB not running
  - seed failure
  - server not reachable
  - timeouts

## Acceptance Notes (for AI-only review)

- The runbook must be precise and command-driven (no ambiguous instructions).
