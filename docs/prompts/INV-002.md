# INV-002: Enhance Inventory View with Product Context

## Implementation Guide

### Problem

After removing Products page (INV-001), users need to see and edit product information inline with inventory items.

### Prerequisites

**INV-001 must be completed first** - removes Products from navigation.

### Current State

The Inventory API already returns product info (`server/inventoryDb.ts`):

- `productName`, `category`, `subcategory`, `brandName`

The `Inventory.tsx` displays this in columns already.

### Enhancements Needed

| Enhancement  | File                                                    | Change                    |
| ------------ | ------------------------------------------------------- | ------------------------- |
| Terminology  | `client/src/pages/Inventory.tsx`                        | "Batch" → "Item"          |
| Product edit | `client/src/components/inventory/BatchDetailDrawer.tsx` | Add edit button           |
| Auto-create  | `server/routers/intakeReceipts.ts`                      | Create product if missing |

### Implementation

**Step 1: Terminology update**

```typescript
// client/src/pages/Inventory.tsx
// Find and replace user-facing labels:
```

