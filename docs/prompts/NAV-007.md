# NAV-007: Add Inbox/Notifications to Main Navigation or Header

## Implementation Guide

### Problem

Users have TWO places to check for action items:

| System           | Table           | Source Types                          | Accessed Via        |
| ---------------- | --------------- | ------------------------------------- | ------------------- |
| `/inbox`         | `inbox_items`   | mention, task_assignment, task_update | Direct URL only     |
| `/notifications` | `notifications` | in_app, email, sms                    | Bell icon in header |

**Verified:** Searched `client/src/config/navigation.ts` - no entry for `/inbox` exists.

Neither is in the main navigation. Users must:

- Click bell icon (notifications only, not inbox)
- Know to type `/inbox` URL directly

### Root Cause

Two parallel notification systems were built independently:

- `notificationService.ts` creates `notifications` records
- `inboxDb.ts` creates `inbox_items` records
- No unified UI to see both

### Solution

Option A (Recommended): Add "Inbox" to navigation with unified view
Option B: Enhance notification bell dropdown to show both sources
