# NAV-007: Add Inbox/Notifications to Main Navigation or Header

## Implementation Guide

### Problem

Users have TWO places to check for action items:

1. `/inbox` - Task assignments, mentions
2. `/notifications` - System notifications

Neither is in the main navigation. Users must:

- Click bell icon (notifications only)
- Know to type `/inbox` URL directly

### Solution

Option A (Recommended): Add unified "Action Items" to navigation
Option B: Make header notification dropdown link to both

### Files to Modify

- `client/src/config/navigation.ts` (for nav option)
- `client/src/components/notifications/NotificationBell.tsx` (for header option)

### Implementation (Option A - Navigation)

```typescript
// Add to Admin group:
{
  name: "Inbox",
  path: "/inbox",
  icon: Inbox,
  group: "admin",
  ariaLabel: "Task assignments, mentions, and action items",
},
```

### Acceptance Criteria

- Users can easily find their action items
- Both inbox and notifications are discoverable
- Badge shows unread count
