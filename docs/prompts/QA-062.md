# QA-062: Implement Sales Sheet Save Functionality

**Priority:** P3 (Low)
**Effort:** 4-6h
**Status:** Not Started

---

## Task Description

Implement the missing save functionality for Sales Sheets. Users can currently create sales sheets, but there is no way to save them.

## Problem

The Sales Sheets feature is incomplete. Without the ability to save, the feature is useless for creating and managing sales materials.

## Acceptance Criteria

- [ ] A "Save" button is added to the sales sheet creation page.
- [ ] Clicking "Save" persists the sales sheet to the database.
- [ ] Saved sales sheets can be viewed and edited.

## Implementation Guide

1.  **Backend:**
    -   Implement a `createSalesSheet` tRPC procedure.
2.  **Frontend:**
    -   Add a "Save" button to the sales sheet creation page.
    -   Connect the button to the `createSalesSheet` procedure.

## Verification

-   Create a sales sheet and click "Save".
-   Verify that the sales sheet is saved and can be accessed later.
