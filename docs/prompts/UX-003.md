# UX-003: Fix Mobile Kanban Overflow

## Implementation Guide

### Problem
In `UnifiedSalesPortalPage.tsx` (lines 351, 843, 942), Kanban columns use `min-w-[300px]` which causes horizontal overflow on mobile devices (< 360px width).

### Current Code
```tsx
<div className="flex-1 min-w-[300px]">
  {/* Column content */}
</div>
```

### Solution
1. Stack columns vertically on mobile
2. Use responsive breakpoints for width
3. Add swipe navigation for mobile kanban

### Files to Modify
- `client/src/pages/UnifiedSalesPortalPage.tsx`

### Implementation
```tsx
// Responsive container
<div className="flex flex-col md:flex-row md:overflow-x-auto gap-4">
  {/* Column with responsive width */}
  <div className="w-full md:flex-1 md:min-w-[300px]">
    {/* Column content */}
  </div>
</div>

// Or use horizontal scroll with snap points on mobile
<div className="flex overflow-x-auto snap-x snap-mandatory md:overflow-visible">
  <div className="snap-center shrink-0 w-[85vw] md:w-auto md:flex-1 md:min-w-[300px]">
```

### Acceptance Criteria
- Kanban usable on mobile devices
- No horizontal overflow on phones
- Columns accessible via scroll or stack
