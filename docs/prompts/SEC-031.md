# SEC-031: Protect /api/auth/push-schema Endpoint

## Problem

The endpoint at `server/_core/simpleAuth.ts:229-243` allows unauthenticated users to push database schema changes. Anyone can modify the database structure remotely.

## Implementation Guide

### Step 1: Add Authentication Middleware

```typescript
app.post("/api/auth/push-schema", authenticateRequest, async (req, res) => {
  // Verify user is admin
  if (!req.user || req.user.role !== "admin") {
    return res.status(403).json({ error: "Admin access required" });
  }
  // ... existing logic
});
```

### Step 2: Add Audit Logging

```typescript
logger.info({
  action: "SCHEMA_PUSH",
  userId: req.user.id,
  timestamp: new Date().toISOString(),
});
```

### Step 3: Add Rate Limiting

```typescript
const schemaLimiter = rateLimit({
  windowMs: 60 * 60 * 1000, // 1 hour
  max: 5 // max 5 schema pushes per hour
});

app.post("/api/auth/push-schema", schemaLimiter, authenticateRequest, ...);
```

## Acceptance Criteria

- [ ] Endpoint requires authentication
- [ ] Only admin users can push schema
- [ ] Audit log entry created for each push
- [ ] Rate limiting prevents abuse
- [ ] Unauthenticated requests return 401
