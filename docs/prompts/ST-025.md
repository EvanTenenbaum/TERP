# ST-025: Add Error Boundaries to Critical Pages

## Implementation Guide

### Problem
Only 2-3 ErrorBoundary usages found in the codebase. 20+ pages lack error boundaries, causing entire page crashes on unhandled errors.

### Affected Pages (Priority)
1. `Inventory.tsx` - Complex async operations
2. `ClientProfilePage.tsx` - Multiple data fetches
3. `Orders.tsx` - Critical workflow page
4. `OrderCreatorPage.tsx` - Complex form with many queries
5. All dashboard widgets in `components/dashboard/widgets-v2/`

### Solution
1. Create PageErrorBoundary wrapper component
2. Apply to all page-level components
3. Add retry and error reporting functionality

### Files to Modify
- Create `client/src/components/common/PageErrorBoundary.tsx`
- Wrap all pages in `client/src/pages/*.tsx`

### Implementation
```typescript
// PageErrorBoundary.tsx
export function PageErrorBoundary({ children }: { children: React.ReactNode }) {
  return (
    <ErrorBoundary
      FallbackComponent={({ error, resetErrorBoundary }) => (
        <div className="p-8 text-center">
          <AlertTriangle className="mx-auto h-12 w-12 text-red-500" />
          <h2 className="mt-4 text-lg font-semibold">Something went wrong</h2>
          <p className="mt-2 text-muted-foreground">{error.message}</p>
          <Button onClick={resetErrorBoundary} className="mt-4">
            Try Again
          </Button>
        </div>
      )}
    >
      {children}
    </ErrorBoundary>
  );
}
```

### Acceptance Criteria
- All critical pages wrapped with ErrorBoundary
- Graceful error display instead of white screen
- Retry functionality available
- Errors logged for debugging
