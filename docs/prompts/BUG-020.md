## Context

Navigating to Todo Lists page returns 404 error. Either the route doesn't exist or the component is missing. Need to investigate if the feature should exist or if the navigation link should be removed.

## Implementation Guide

### Step 1: Investigate Current State

Check for todo-related code:

- Search for todo routes in `client/src/App.tsx`
- Search for todo routers in `server/routers/`
- Check sidebar navigation for todo link in `client/src/components/`

```bash
grep -r "todo" client/src --include="*.tsx" --include="*.ts"
grep -r "todo" server/routers --include="*.ts"
```

### Step 2: Decision Point

Based on investigation:

- **If backend exists**: Implement TodoListsPage component
- **If backend doesn't exist**: Remove navigation link from sidebar

### Step 3A: If Implementing TodoListsPage

Create `client/src/pages/TodoListsPage.tsx` with:

- List of todo lists
- Create new todo list functionality
- Navigation to individual todo list
- Loading and empty states

Add route in `client/src/App.tsx`:

```tsx
<Route path="/todos" component={TodoListsPage} />
```

### Step 3B: If Removing Navigation

Update sidebar navigation in `client/src/components/DashboardLayout.tsx` or similar:

- Remove or hide the Todo Lists navigation link
- Ensure no dead links remain

### Step 4: Test

- Verify no 404 errors on navigation
- Test todo functionality (if implemented)
- Verify TypeScript types

## Success Criteria

- [ ] No 404 error when clicking Todo Lists (either page works or link removed)
