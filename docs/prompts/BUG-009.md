## ðŸŽ¯ Context

**Problem:**
Navigating to `/create-order` returns a 404 Page Not Found error.

**Impact:**
Users cannot create orders via the direct route, disrupting the workflow.

**Goal:**
Restore the `/create-order` route or redirect it to the correct order creation interface.

---

## Phase 1: Investigation

### Step 1.1: Check Routing
1. Open `client/src/App.tsx`.
2. Search for `/create-order`.
3. Check if the route exists and points to a valid component.
   ```tsx
   <Route path="/create-order" component={CreateOrderPage} />
   ```

### Step 1.2: Check Component
1. Verify if `CreateOrderPage.tsx` exists in `client/src/pages/`.
2. If not, check if it was renamed or moved.

### Step 1.3: Check Permissions
1. Ensure the route isn't guarded by a permission the user doesn't have.

---

## Phase 2: Fix

### Step 2.1: Restore/Redirect Route
- **If route is missing:** Add it back to `App.tsx`.
- **If component is missing:** Find the correct component (maybe `OrderFormPage`?) and wire it up.
- **If path changed:** Update the route or add a redirect.

**Recommendation:**
If there is a centralized "Create" flow (e.g. `/orders/new`), consider redirecting `/create-order` to that, or aliasing it.

### Step 2.2: Verify Sidebar Links
1. Check `client/src/components/layout/AppSidebar.tsx`.
2. Ensure the "Create Order" link points to the correct, working URL.

### Step 2.3: Verify
1. Navigate to `/create-order`.
2. Verify the order creation form loads.
