# BUG-074: Add isPending Checks to ClientProfilePage Mutations

## Implementation Guide

### Problem
In `ClientProfilePage.tsx` (lines 115-128), `updateClientMutation`, `createTransactionMutation`, and `recordPaymentMutation` don't have visible `isPending` checks on submit buttons, enabling double-submit.

### Current State
- Submit buttons not disabled during mutation
- Multiple rapid clicks can create duplicate records
- No loading indicator during submission

### Solution
1. Add `disabled={mutation.isPending}` to all submit buttons
2. Add loading spinner during submission
3. Consider using useAppMutation hook for consistency

### Files to Modify
- `client/src/pages/ClientProfilePage.tsx`

### Implementation
```typescript
<Button
  type="submit"
  disabled={updateClientMutation.isPending}
>
  {updateClientMutation.isPending ? (
    <>
      <Loader2 className="mr-2 h-4 w-4 animate-spin" />
      Saving...
    </>
  ) : (
    'Save Changes'
  )}
</Button>
```

### Acceptance Criteria
- All mutation buttons disabled during pending state
- Visual loading indicator shown
- No duplicate submissions possible
