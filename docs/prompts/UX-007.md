# UX-007: Fix Sidebar Skeleton Responsive Width

## Implementation Guide

### Problem
In `DashboardLayoutSkeleton.tsx` (line 7), the sidebar skeleton uses `w-[280px]` without mobile fallback, causing layout mismatch during loading.

### Current Code
```tsx
<div className="w-[280px] border-r">
  {/* Skeleton content */}
</div>
```

### Solution
Match actual sidebar responsive behavior: hidden on mobile, fixed width on desktop.

### Files to Modify
- `client/src/components/DashboardLayoutSkeleton.tsx`

### Implementation
```tsx
// Match actual Sidebar responsive behavior
<div className="hidden md:block w-[280px] border-r">
  {/* Skeleton content */}
</div>

// Or use same classes as real Sidebar
<div className="fixed inset-y-0 left-0 z-50 w-64 -translate-x-full md:relative md:translate-x-0 md:w-64 border-r">
  {/* Skeleton content */}
</div>
```

### Acceptance Criteria
- Skeleton matches actual sidebar responsive behavior
- No layout shift between loading and loaded states
- Mobile view shows appropriate skeleton
