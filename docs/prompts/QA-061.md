# QA-061: Fix Returns Processing Modal

**Priority:** P1 (High)
**Effort:** 8-12h
**Status:** Not Started

---

## Task Description

Fix the broken modal for processing returns. The modal currently fails to display the line items from the selected order, blocking the entire returns workflow.

## Problem

Customer service representatives cannot process returns, which is a critical business function. This leads to poor customer satisfaction and manual workarounds.

## Acceptance Criteria

- [ ] The "Process Return" modal correctly displays all line items from the selected order.
- [ ] Users can select items and specify quantities to be returned.
- [ ] A return merchandise authorization (RMA) is created upon submission.

## Implementation Guide

1.  **Backend:**
    -   Debug the tRPC procedure that fetches order line items for the return modal.
2.  **Frontend:**
    -   In the return modal component, ensure the data is correctly fetched and displayed.
    -   Fix any issues with the item selection and quantity input fields.

## Verification

-   Select an order and open the "Process Return" modal.
-   Verify that all order line items are displayed.
-   Select an item to return and submit the form. Verify that an RMA is created.
