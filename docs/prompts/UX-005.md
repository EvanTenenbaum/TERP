# UX-005: Make Select Dropdowns Responsive

## Implementation Guide

### Problem
Many select dropdowns use fixed widths like `w-[180px]` that don't adapt to mobile screens, causing overflow or cramped UI.

### Affected Components
- `BulkActionsBar.tsx` - `w-[180px]`
- `pagination-controls.tsx` - `w-[70px]`
- `bulk-actions.tsx` - `w-[160px]`
- `Bills.tsx` - `w-[200px]`
- `BankTransactions.tsx` - `w-[180px]`
- Multiple dashboard widgets with `w-[140px]`

### Solution
Apply responsive width pattern: `w-full sm:w-[180px]`

### Files to Modify
- All files listed above

### Implementation
```tsx
// BEFORE
<SelectTrigger className="w-[180px]">

// AFTER
<SelectTrigger className="w-full sm:w-[180px]">
```

### Good Example (Already Implemented)
```tsx
// From Invoices.tsx - correct pattern
<SelectTrigger className="w-full sm:w-[200px]">
```

### Acceptance Criteria
- All select dropdowns full-width on mobile
- Constrained width on desktop
- No horizontal overflow on mobile
