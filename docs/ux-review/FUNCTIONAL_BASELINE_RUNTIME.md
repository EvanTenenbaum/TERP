# TERP Functional Baseline — Runtime UX Supplement

> Generated by live browser walkthrough of staging (`https://terp-staging-yicld.ondigitalocean.app`)
> Date: 2026-04-22
> Companion to: `docs/FUNCTIONAL_BASELINE.md`
> Capture method: `agent-browser` (Playwright-driven) against staging in DEMO_MODE auto-login as **QA Super Admin**.
> Scope: Browse only — no records were created, edited, deleted, or submitted.

---

## Summary

This document records **what each TERP surface actually looks and behaves like at runtime** on staging with realistic seed/production-like data. It is the as-observed complement to `docs/FUNCTIONAL_BASELINE.md` (which inventories routes/components/business rules from source). Sections below are organized per the walkthrough checklist and each describes the actual URL visited, visible data, empty/loading/conditional states, tabs, actions, and anything role-specific.

Global observations:

- **Shell chrome is persistent across every protected page**: left sidebar (collapsible) with top "Sales Quick Actions" tiles, then expandable groups `Sell`, `Buy`, `Operations`, `Relationships`, `Finance`, `Admin`. Top bar has a global search (`Ctrl+K`), notification bell with a `9+` badge, account button (shows `QA Super Admin`), and a "U" / "Menu" affordance.
- **Breadcrumbs** appear directly under the top bar (e.g. `Home ▸ Sales`, `Home ▸ Inventory`).
- **Every workspace uses the same two-row "workspace header"**: bold title, one-line subtitle describing the purpose, then a **blue "Primary / Current view / Handoff" band** with tab chips.
- **Spreadsheet-native surfaces** consistently show a "Spreadsheet View / Standard View" toggle at the top of the content area, a column header strip, and a footer reading "Click select row · Shift+Click extend range · Ctrl+Click add to selection · Ctrl+C copy cells · Ctrl+A select all".
- **A staging banner `TERP [STAGING]`** is rendered as the product wordmark in the sidebar header. Production would presumably drop the `[STAGING]`.
- **An annotation overlay toolbar** (v3.0.2) is embedded on every page — this is a dev/QA capture tool (labels: Pause animations, Layout mode, Hide markers, Copy feedback, Send Annotations, Clear all, Settings, Output Detail, React Components, Marker Color, Block page interactions, Manage MCP & Webhooks). It is not part of the core TERP UI but is always present on this build.
- **Loading states** are generally terse — a plain `Loading...` string (e.g. `/clients/:id/vip-portal-config`) or no skeleton at all; most list surfaces render content or an explicit empty state directly.
- **A global "Update Available 🚀"** banner appeared once at the top of `/notifications` reading "A new version is available. Click to reload." with a `Reload` button — this is an app-shell update notifier that fires when a new bundle is deployed.

## Global Conditional UI & Messages Observed (catalog)

All strings below are verbatim copy that the product surfaces conditionally based on data/state. A UI redesign must preserve these (or explicit replacements):

- **`Update Available 🚀 · A new version is available. Click to reload. · Reload`** — shell-level banner that appears after a new bundle ships (seen once on `/notifications`).
- **`Unable to load daily snapshot — Transaction or inbox data could not be loaded`** — dashboard "Today at a Glance" degraded state when the backend call errors.
- **`You're short $<amount> after scheduled bills.`** — conditional on `Available After Bills < 0` (Cash Position card).
- **`You owe suppliers $<amount> more than customers owe you — collect faster.`** — conditional on `net position < 0` (Money In vs. Out card).
- **`All suppliers are paid up. No sold-out batches with outstanding balances.`** — conditional empty for the "Suppliers Waiting to Be Paid" card.
- **`No upcoming appointments — No appointments scheduled for the next 7 days.`** — Appointments card empty state.
- **`N/A vs last week`** — Cash Collected 7d delta when previous week is `$0`.
- **`<N> overdue invoices need attention · The overdue queue is above the alert threshold of 25. Work oldest items first or open the invoice queue now.`** — accounting dashboard alert when overdue count exceeds 25 (observed: 371).
- **`<N> items need attention · <M> AR payments without matching invoices · <K> AP payments without matching bills`** — reconciliation anomalies aggregate on accounting dashboard.
