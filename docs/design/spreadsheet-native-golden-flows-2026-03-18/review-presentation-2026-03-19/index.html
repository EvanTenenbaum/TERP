Review presentation

TERP spreadsheet-native proposed designs

This deck is the current proposed design set for all spreadsheet-native pages after the latest TERP reality audit, the Gemini deep QA pass, and the final polish patch round.

What changed in this round

  • Sales Sheet search language was standardized from vendor to supplier.
  • The shared status-bar primitive now includes a visible Keys / Help trigger for keyboard discoverability.
  • The presentation itself is structured for comment-ready review in Figma or chat.

How to review

  • Comment on clarity, confidence, density, and workflow trust, not just visual taste.
  • Call out what should stay, what should change, and why the current version misses the mark.
  • Use the comment tag format shown on every slide so feedback is easy to apply quickly.

What strong feedback looks like

  • Specific location: name the page and the panel, strip, card, or grid area.
  • Specific issue: say whether it is layout, copy, interaction, hierarchy, or trust.
  • Specific outcome: explain what you want the operator to understand or do faster.
