# TER-574 - Photography Complete Flag Migration (enum to boolean)

## Scope

- Removed `PHOTOGRAPHY_COMPLETE` from the `batchStatusEnum` in schema
- Added `isPhotographyComplete` INT boolean column to batches table
- Migration SQL: expand enum → migrate data → shrink enum
- Updated all server code (routers, services, queries) to use boolean flag
- Updated all client code (components, types, status lists) to remove the status
- Updated seeds, generators, and test files
- Photography workflow now sets `isPhotographyComplete=1` instead of changing batch status

## Rollback Plan

```sql
-- Step 1: Re-add PHOTOGRAPHY_COMPLETE to the enum
ALTER TABLE `batches` MODIFY COLUMN `batchStatus` enum('AWAITING_INTAKE','LIVE','PHOTOGRAPHY_COMPLETE','ON_HOLD','QUARANTINED','SOLD_OUT','CLOSED') NOT NULL DEFAULT 'AWAITING_INTAKE';
-- Step 2: Restore status for batches that had photography complete
UPDATE `batches` SET `batchStatus` = 'PHOTOGRAPHY_COMPLETE' WHERE `isPhotographyComplete` = 1 AND `batchStatus` = 'LIVE';
-- Step 3: Drop the boolean column
ALTER TABLE `batches` DROP COLUMN `isPhotographyComplete`;
```
