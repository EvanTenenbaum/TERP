# TER-572 - Product Metadata to Settings + Batch Drawer Editing

## Scope

- Moved product management into `Settings > Master Data > Product Metadata`
- Removed the Inventory workspace `Products` tab
- Removed the Inventory workspace `Jump to Products` CTA
- Added batch-drawer editing for linked product metadata:
  - `name`
  - `category`
  - `strain`
  - `grade`
- Redirected legacy `/products` navigation to `/settings?tab=product-metadata`

## Repo Reality Notes

- Current schema stores `grade` on the batch record, not on the product record. TER-572 therefore updates:
  - product `name/category/strain` through `productCatalogue.update`
  - batch `grade` through `inventory.updateBatch`
- Current schema marks `batches.productId` as non-null, but the batch drawer still guards the missing-link case and blocks editing with an explicit UI state instead of assuming impossible data.
