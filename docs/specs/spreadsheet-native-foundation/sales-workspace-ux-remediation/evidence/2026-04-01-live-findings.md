## Quick Wins

- F-004: The empty state leaves a large blank canvas and says only 'Select a client to start building a catalogue', while the actual next action is hidden up in the toolbar. The page looks inactive and under-explained. -> Replace the passive empty state with an anchored setup card near the top of the canvas that explicitly guides the user through choosing a client, naming the draft, and then adding the first row.
- F-001: The supplied URL includes `ordersView=document`, but because `tab=sales-sheets` wins, users land on the Sales Catalogue surface instead of an order-document view. The URL communicates one intent while the page renders another. -> Normalize or redirect incoherent sales URLs so any `ordersView=document` intent resolves onto `tab=orders` or `tab=create-order`, and never silently renders Sales Catalogue when the document intent is present.
- F-003: On mobile the tab strip truncates labels and the sheet/classic toggle is visibly clipped, creating immediate navigation friction before the user can even start the task. -> Make the command strip stack vertically on mobile and let the mode toggle buttons wrap or fill separate rows; keep the tabs horizontally scrollable but preserve full label readability and stronger active-state cues.
- F-002: Orders and Create Order both route users into nearly the same sheet-native order composer, so the top-level IA suggests two different destinations while the experience feels like one duplicated surface. -> Clarify the relationship between `Orders` and `Create Order` by making one the queue and one the composer with visibly different headers, or collapse them into one orders workspace with explicit queue/composer states.
- F-006: The sheet-native order composer opens with a lot of empty space, disabled save/finalize controls, and very little contextual guidance, which makes the surface feel unfinished rather than intentionally staged. -> Use the initial document area to show a concise start checklist and surface the required first decision inline, then reserve the large spreadsheet canvas for after a customer or draft context exists.

## Less Busy Opportunities

- F-005: Consolidate setup and conversion actions into one progressive control area: primary setup actions up top, advanced view controls only after a client is selected, and conversion/share actions promoted only when the sheet becomes eligible.
