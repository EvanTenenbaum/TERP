# Documentation Archive

This directory contains historical documentation from the TERP project. These files are preserved for reference but are no longer actively maintained.

## Archive Structure

### 2025-11/
Current month's archived documentation, organized by category:

- **qa-reports/** - Quality assurance reports from various features and phases
- **delivery-packages/** - Completed feature delivery packages and summaries
- **specifications/** - Old feature specifications and design documents
- **implementation-guides/** - Historical implementation plans and roadmaps
- **testing-reports/** - Test reports and investigation documents
- **analysis/** - Codebase analysis and strategy documents

### Task-Specific Archives (Added 2025-11-13)

- **completion-reports/** - Task completion reports and final status documents
- **guides/** - Historical guides and procedural documentation
- **investigation-reports/** - Bug and issue investigation reports
- **planning/** - Historical planning documents and migration plans
- **patterns/** - Historical implementation patterns
- **fixes/** - Historical bug fix documentation

### Feature-Specific Archives

- **calendar/** - Calendar module evolution specs and historical docs (V3.0-V3.2)
- **vip-portal/** - VIP Client Portal feature specs and implementation docs
- **quote-sales/** - Quote and Sales module specifications and workflows
- **misc/** - Other historical documentation (handoffs, migrations, deployment guides, etc.)

## Accessing Archived Documentation

All archived files are still accessible via Git and can be referenced when needed. Use the directory structure above to locate specific historical documents.

## Archive Policy

Documents are archived when they:
- Represent completed work (delivery packages, QA reports)
- Are superseded by newer versions
- Are no longer actively referenced
- Represent historical context rather than current state
