Layout Zones (top → bottom)

1. Toolbar Title + KPI badges (AR total, overdue count, total invoices) + AR Aging toggle + search + refresh
2. Action Bar Status filter tabs (All/Draft/Sent/Viewed/Partial/Paid/Overdue/Void) + workflow actions (Create, Mark Sent, Record Payment, Void, PDF, Print)
3. Grid + Inspector PowersheetGrid (read-only, full-width by default) + collapsible InspectorPanel (~25% right, opens on row select, Esc to close)
4. AR Aging Panel Collapsible. 5 aging buckets (Current / 1-30 / 31-60 / 61-90 / 90+) with amounts and counts. From invoices.getARAging.
5. Client Ledger Sub-View Collapsible mini-grid. Appears when a row is selected. Shows selected client's transaction history (date, type, description, debit, credit, running balance). Actions: Add Adjustment, Export CSV, Date range filter. Paginated (50 rows, [/] shortcuts).
6. Status Bar Selection count + active filter + total count + ledger context | KeyboardHintBar

Key Behaviors

  • Inspector opens on row select — grid animates from 100% to ~75% width. Close with ✕ or Esc.
  • Client Ledger auto-populates — selecting any invoice row loads that client's ledger below. Different client = new ledger data.
  • AR Aging is opt-in — collapsed by default, toggled from toolbar button. Persists toggle state in localStorage.
  • Workflow actions are context-sensitive — buttons enable/disable based on selected row's status (e.g., can't Void a PAID invoice, can't Mark Sent a VOID invoice).
  • InvoiceToPaymentFlow golden flow — "Record Payment" opens the existing sidecar dialog (941 lines, preserved as-is).
  • Create Invoice dialog — existing dialog from pilot, uses accounting.invoices.generateNumber.
  • Void with reason dialog — existing dialog from pilot, calls invoices.void with GL reversal.
  • Deep-link support — ?invoiceId= selects that row and opens inspector on mount.
  • Client name is clickable — navigates to client detail (or opens drawer).
