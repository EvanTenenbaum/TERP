Core Registry Layout: 3 Approaches

How should Invoices, Bills, and Payments surfaces be composed?
These are browse+act registries, not build-a-document surfaces like Sales Order.

A

Full-Width Grid + Toggling Inspector

Grid takes 100% width. Inspector slides in from right when a row is selected (pushing grid narrower). Support modules (KPI band, aging, Client Ledger) stack below.

Pros

  • Maximum grid real estate for scanning
  • Inspector only appears when needed
  • Simplest — no split to manage

Cons

  • Inspector toggle causes layout shift
  • No persistent detail view
B

Grid + Persistent Inspector Rail (75/25)

Grid takes ~75%, inspector always visible at ~25% right. Inspector shows detail of selected row. Support modules below full width. This is what the current pilots already do.

C

Grid + Collapsible Inspector (Recommended)

Grid takes 100% by default. Selecting a row opens inspector at ~25% with smooth animation. Close button collapses it back. Best of both worlds. KPI badges in toolbar (not a separate band) to save vertical space.
