Component Architecture — OrderCreatorPage Decomposition

Zone Extract From New Component

Key Design Decisions

  • Left grid is read-only PowersheetGrid (inventory browse + add action). Right grid is editable PowersheetGrid (reuses OrdersDocumentLineItemsGrid with fill handle + undo/redo).
  • Inventory grid is shared between both surfaces — InventoryBrowserGrid wraps PowersheetGrid with column defs and data source. Sales Catalogue passes client-priced data, Sales Order passes batch/SKU-level data.
  • Support modules are always visible below the split — not in the InspectorPanel. Referral, Credit, and Whole Order Changes are compact inline cards per mockup direction.
  • Filtering + saved views work identically to Sales Catalogue — same AdvancedFilters, QuickViewSelector, SaveViewDialog components, scoped to the selected customer.
  • State extracted to useOrderDraft() — autosave (2s debounce), fingerprinting, dirty tracking, draft CRUD, finalize. Replaces inline state from OrderCreatorPage.
  • Retire: OrderCreatorPage (21K), OrdersSheetPilotSurface document mode, SheetModeToggle for orders/create-order tabs.
  • OrdersSheetPilotSurface queue mode preserved — the orders queue grid (list of all orders) stays as-is. Only document/create mode is replaced.
