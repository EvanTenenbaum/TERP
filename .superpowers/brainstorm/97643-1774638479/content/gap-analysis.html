Payment Terms — New Feature Flag

The audit confirmed Payment Terms is NOT in the current OrderCreatorPage. This is genuinely new work. I recommend scoping it as:

  • UI: Select dropdown in invoice-bottom (Net 15 / Net 30 / Net 45 / Net 60 / COD / Prepaid) + "★ Save default" button
  • Schema: Add paymentTerms column to orders table + defaultPaymentTerms to clients table
  • Server: Include in createDraftEnhanced/updateDraftEnhanced/finalizeDraft payloads
  • Default: Auto-populate from customer profile on client selection, editable per order
  • Flag it separately in the plan so it can ship independently if needed
