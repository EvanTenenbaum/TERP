Layout Flow (top → bottom)

Zone What Why
1. Toolbar Context: customer, doc type, seed badge, autosave, Save/Finalize Always visible — trust cues at a glance
2. Action bar Row ops (Add/Dup/Del), Filters, Saved Views, running total Grid-scoped actions near the grids
3. Split grids Inventory (left 40%) | Order Document (right 60%) Side-by-side browse + build — the work surface
4. Invoice bottom Subtotal → Discount → Freight → Total, then Payment Terms + Credit Anchored to right grid — reads like the bottom of a real invoice. Terms are saveable as customer default.
5. Order Adjustments Referral, Notes, Margin Override, Draft status Full-width below split — order-level settings, not line-level
6. Status bar Context summary + keyboard hints Persistent footer

Key Changes from V1

  • Visual break — 6px gap + rounded borders separate inventory and order grids. They're distinct cards, not flush panels.
  • Invoice bottom — right-aligned inside the order grid card: Subtotal, Discount, Freight, Total. Reads like the bottom of a real invoice.
  • Payment terms — selector with ★ Save default button. Editable accent border (same as grid editable cells). Defaults persisted to customer profile.
  • Credit info — subtle inline display: available amount + usage % warning. Not a separate card — it lives next to terms because they're related (terms affect credit exposure).
  • Referral moved to Order Adjustments — it's an order-level setting, not a trust-critical summary. Compact card in the 4-column adjustments bar alongside Notes, Margin, and Draft status.
  • Whole Order Changes → "Order Adjustments" — simpler name, full-width bar with 4 compact cells.
