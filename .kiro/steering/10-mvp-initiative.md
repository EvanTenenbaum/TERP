### When Asked to Define a Sprint

**ALWAYS** pull tasks from the current MVP phase:

```
1. Read docs/roadmaps/MVP_ROADMAP.md
2. Identify current phase
3. Select available tasks from that phase
4. Check for dependencies
5. Assign based on parallelization strategy
```

### When Asked for Roadmap Updates

**ALWAYS** update the MVP Roadmap:

```
1. Update task status in MVP_ROADMAP.md
2. Update phase progress percentage
3. Add status update entry to log
4. If phase complete, note checkpoint needed
```

### Deviation Protocol

If a task is NOT on the MVP roadmap:

1. **Ask:** Is this a critical production bug?
2. **Ask:** Does this unblock an MVP task?
3. **If NO to both:** Do NOT prioritize it
4. **If YES:** Get explicit user approval before proceeding
