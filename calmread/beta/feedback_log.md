# Beta Feedback Log

**Version:** 1.0.0  
**Last Updated:** January 11, 2026  
**Purpose:** Track observations and feedback from beta testing

## How to Use This Log

1. Record observations during or immediately after each beta session
2. Use one entry per observation
3. Include as much context as possible
4. Tag entries for easy filtering
5. Review weekly to identify patterns

## Entry Template

```markdown
### Entry [XXX]

**Date:** YYYY-MM-DD
**Session:** [session_id]
**Observer:** [parent/tester name]
**Child Age:** [age in years/months]
**Lesson:** [lesson_id]
**Device:** [device model]

**Category:** [usability | engagement | technical | content | calm_design | other]
**Severity:** [blocker | critical | major | minor | observation]

**Observation:**
[Describe what was observed]

**Context:**
[What was happening before/during/after]

**Child Reaction:**
[How did the child respond - behavior, verbal, emotional]

**Suggested Action:**
[What might help, if any ideas]

**Tags:** [tag1, tag2, tag3]

---
```

## Categories

| Category | Description | Examples |
|----------|-------------|----------|
| usability | UI/UX issues | Can't find button, confusing navigation |
| engagement | Attention/interest | Lost interest, very engaged, distracted |
| technical | Bugs, crashes | App crashed, audio didn't play |
| content | Lesson content | Word too hard, story confusing |
| calm_design | Calm violations | Seemed overstimulated, wanted to stop |
| other | Anything else | General observations |

## Severity Definitions

| Severity | Definition | Action |
|----------|------------|--------|
| blocker | Prevents use | Fix immediately |
| critical | Significantly impacts use | Fix before next session |
| major | Noticeable problem | Fix soon |
| minor | Small issue | Track for future |
| observation | Neutral note | Record for patterns |

## Tags

Use these standard tags for filtering:

### Behavior Tags
- `engaged` - Child was engaged
- `distracted` - Child was distracted
- `frustrated` - Child showed frustration
- `confident` - Child showed confidence
- `struggling` - Child was struggling
- `completed` - Completed the lesson
- `stopped_early` - Stopped before completion

### Content Tags
- `too_easy` - Content was too easy
- `too_hard` - Content was too hard
- `just_right` - Difficulty appropriate
- `confusing` - Content was confusing
- `engaging_story` - Story was engaging
- `boring` - Content was boring

### Technical Tags
- `audio_issue` - Audio problem
- `touch_issue` - Touch response problem
- `display_issue` - Display problem
- `crash` - App crashed
- `slow` - Performance issue

### Calm Design Tags
- `calm` - Child remained calm
- `overstimulated` - Signs of overstimulation
- `wanted_more` - Wanted to continue (check if compulsive)
- `natural_stop` - Stopped naturally at endpoint
- `peaceful_end` - Ended session peacefully
