# Beta Session Protocol

**Version:** 1.0.0  
**Last Updated:** January 11, 2026  
**Purpose:** Guide parents/testers through consistent beta testing sessions

## Overview

This protocol ensures consistent, valuable beta testing while prioritizing the child's experience and wellbeing. Follow these steps for each beta session.

## During the Session

### Starting the Session

1. **Introduce casually**
   - "Let's try this reading app together"
   - No pressure, no expectations stated
   - Child can stop anytime

2. **Position for observation**
   - Sit beside, not behind
   - Can see screen and child's face
   - Don't hover

3. **Start the app**
   - Let child tap to start if possible
   - Provide minimal guidance initially

### Observation Guidelines

**DO observe:**
- Facial expressions
- Body language
- Verbal comments
- Touch interactions
- Attention/focus
- Emotional state
- Natural stopping points

**DO NOT:**
- Interrupt unnecessarily
- Provide answers
- Express judgment
- Rush the child
- Compare to other children
- Praise excessively
- Show disappointment

### Intervention Triggers

**Intervene if:**
- Child asks for help
- Child is visibly frustrated (> 30 seconds)
- Technical issue occurs
- Child wants to stop
- Safety concern

**How to intervene:**
- Calm, supportive tone
- "Would you like some help?"
- "It's okay, let's try together"
- "We can stop if you'd like"

### Ending the Session

**Natural end:**
- Child reaches "All Done" screen
- Note: Did they seem satisfied?
- Note: Did they want to continue?

**Child-initiated end:**
- Child asks to stop
- Note: At what point?
- Note: What was their state?
- Respond: "Okay, that's fine. Good reading!"

**Parent-initiated end:**
- Only if necessary (time, behavior)
- Note: Why ended early?
- Find natural pause point if possible

## After the Session

### Immediate Recording (within 5 minutes)

Complete these while fresh:

1. **Session metadata**
   - Date and time
   - Duration
   - Lesson(s) attempted
   - Completion status

2. **Overall impression**
   - Child's general engagement
   - Emotional arc (start ‚Üí middle ‚Üí end)
   - Any standout moments

3. **Specific observations**
   - Transfer notes to feedback log
   - Add context and detail
   - Include exact quotes

### Child Debrief (optional, brief)

If child is willing, ask casually:
- "What did you think?"
- "Was there anything hard?"
- "Was there anything fun?"

**Do NOT ask:**
- Leading questions
- "Did you like it?"
- Anything that implies expected answer

## Observation Focus Areas

### Week 1-2: Usability Focus

Primary observations:
- Can child navigate independently?
- Are touch targets adequate?
- Is text readable?
- Are instructions clear?

### Week 3-4: Content Focus

Primary observations:
- Is difficulty appropriate?
- Are words decodable?
- Is story engaging?
- Are comprehension questions clear?

### Week 5-6: Calm Design Focus

Primary observations:
- Does child remain calm?
- Is endpoint satisfying?
- Any signs of overstimulation?
- Natural stopping behavior?

### Ongoing: Technical Focus

Always note:
- Any crashes or freezes
- Audio issues
- Touch response issues
- Display problems

## Safety Guidelines

### Child Wellbeing First

- **Stop if child is distressed**
- **No forcing participation**
- **Respect "no" and "stop"**
- **Keep sessions short (10-15 min max)**
- **Positive experience priority over data**

### Data Privacy

- No photos/videos of child
- No sharing child's name externally
- Observations are confidential
- Data used only for app improvement

### Technical Safety

- Kiosk mode prevents app escape
- No internet access
- No external links
- No in-app purchases

## Troubleshooting

### Common Issues

| Issue | Solution |
|-------|----------|
| App won't start | Restart device, try again |
| Audio not playing | Check volume, check speaker |
| Touch not responding | Clean screen, restart app |
| Lesson not loading | Check lesson files exist |
| Recording not working | Check microphone permission |

### When to Stop Testing

- Device malfunction
- Child refuses to continue
- Repeated crashes
- Child shows distress
- Time constraint

### Reporting Critical Issues

If you encounter:
- Data loss
- Repeated crashes
- Harmful content
- Safety concern

Report immediately via [contact method].
